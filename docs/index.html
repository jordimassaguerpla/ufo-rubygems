Ruby gems security update frequency


Ruby gems get updated, but how often? And why it matters? Knowing the frequency of the updates can help us predict how much effort will mean to maintain a ruby application which requires a given set of gems.

In order to have a good prediction, we need to look at the history and how this data looks like.

In chart#1 we can see a plot where the x-axis is the year and the y-axis is the number of security updates. We have chosen to look at the security ones cause that is when you MUST update. To be fair, we are looking at the security issues, and we are assuming each security issue means an update.

We deliberately choose ignoring other updates, as it is what you will usually do, unless they fix a bug which fixes an issue on your app.

You can select and unselect a different set of gems. Use that tool to help you decide which gems should you use on your project and have an idea on the number of times you may have to update them.

Data for this project comes from the ruby-advisory-db project.
